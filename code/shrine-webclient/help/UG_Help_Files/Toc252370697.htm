1.4 Constructing a Query

SHRINE enables you to explore your topic in whatever depth is available currently, as long as your individual queries are related to your general Query Topic.

To query SHRINE data:

  1. Point your browser to the address identified when you registered for SHRINE, and log in by using your institution account credentials.
    Note: You must be behind your hospital's firewall to view the login page.
    When you are logged in to SHRINE, you see the SHRINE system in your browser.
  2. Select a Query Topic.
    In the Query Tool view, use the Select a Topic pull-down list in the middle-right side of the window and select the topic to which this query applies.
    The first time that you use the Query Tool, you will not see any data listed in the Terms view until you select a Query Topic.
  3. Navigate (browse) or find (search) for medical terms from which you choose to query for a patient cohort.
    In the Terms pane, click the Navigate Terms tab to browse for patient characteristics or medical terms that you would like to query. Click the Expand (+) button for a category to view criteria subcategories.
    To locate a specific term quickly, use the Find Terms tab options.
    Terms comprise the following characteristics:
    • Demographics
    • Diagnoses
    • Medications
    • Laboratory values
    See the data reference for more detailed information about the types of data available for you to query.
  4. Drag your selected term to the Query Tool view, Group 1 field.
    Tip: To optimize the rate at which your query executes, select the most-limiting terms first. For example, demographic data is broader in scope than diagnoses or lab values. In addition, lab test terms take longer to process than other terms. Therefore, if you select your diagnoses terms first and then select your lab or demographic terms, SHRINE returns your query results more quickly.
    See the Query Quick Reference for detailed information about how to optimize query formats and define query logic.
  5. Right-click your selected term in the Group field to select patients with specific values of that term.
    Or, apply constraints to all terms in a group by using the Date, Occurs, and Exclude options at the top of the Group.
  6. Repeat steps 3—5 for additional terms to narrow your query.
    You can add terms within a Group to apply OR logic, and use multiple Groups to define your selection by using AND logic.
  7. When your query terms are defined, click Run Query.
    You are prompted to enter a name for this query. The default name is a string derived from the terms that you selected for the query and the time at which you run it.
  8. Enter a name for the query or accept the default, and then click OK to run your query.
    When the query process completes, the aggregate number of potential patient cohorts is displayed in the Query Status view.
    Note: You cannot access patient records through SHRINE.
